### Reward Coefficients

| Parameter | Value | Source |
| --- | --- | --- |
| `alignment_rew_coef` | `1.0` | diff-dataset G1 script |
| `diversity_rew_coef` | `1.0` | diff-dataset G1 script |
| `ce_loss_coef` | `0.03` | diff-dataset G1 script |

Reward 公式：

```text
alignment = 2 * cosine_similarity(gen_embedding, gt_embedding)
diversity = 2 * mean_inner_product_with_other_samples(gen_embedding)
reward = alignment_rew_coef * alignment - diversity_rew_coef * diversity
advantage = reward - RLOO_baseline
```

### Strided Block Geometry

| Parameter | Value |
| --- | --- |
| `prompt_max_len` | `384` |
| `context_max_len` | `8` |
| `generate_max_len` | `8` |
| `stride` | `8` |
| `n_samples_per_prompt` | `4` |

Derived:

```text
num_blocks = (prompt_max_len - generate_max_len - context_max_len) / stride + 1
           = (384 - 8 - 8) / 8 + 1
           = 47

tokens generated by strided blocks per sample = num_blocks * generate_max_len
                                               = 47 * 8
                                               = 376
```

Required invariants:

- `generate_max_len < prompt_max_len`
- `prompt_max_len >= generate_max_len + context_max_len`
- `(prompt_max_len - generate_max_len - context_max_len) % stride == 0`
- `micro_train_batch_size >= n_samples_per_prompt`
- `micro_train_batch_size % n_samples_per_prompt == 0`
- `micro_rollout_batch_size % n_samples_per_prompt == 0`
- `train_batch_size % (micro_train_batch_size * actor_gpus) == 0`

### Actor Loss Note

OpenRLHF G1 使用 `EBFTPolicyLoss`：

- RL 部分作用于 generation/action tokens。
- CE 部分作用于非 action 且 QA mask 允许的区域。
- diff-dataset G1 脚本中 `ce_loss_coef` 默认是 `0.03`。

当前 slime 已接入 G1 token advantages，但仍走 PPO-style policy loss。这一点不是 Step 0 阻塞项，但必须作为 Step 5 的明确决策点。

## Execution Rules

后续执行必须遵守以下顺序：

1. slime baseline 已由用户确认通过，不再阻塞 G1。
2. G1 math parity 已通过固定 embedding 与 helper-level dump 检查。
3. 再补 rollout embedding。
4. 再跑 G1 小闭环。
5. 再决定是否补 EBFT actor loss。
6. 最后才进入加速实验。

任何时候如果出现训练不稳定或指标异常，优先回退到最近一个已通过的阶段，不同时修改多个变量。

Step 0 可以视为通过，当以下事项都满足：

- G1 scope 已固定为 pointwise / single / no teacher。
- G2/G3、CF、OPD、async 明确排除在当前阶段之外。
- OpenRLHF G1 的关键参数已列出。
- strided block 几何已计算并记录为 `num_blocks=47`。
- slime 当前已有能力和缺口已列出。
- 后续第一执行项明确为 slime baseline 和 G1 math parity，而不是加速。

当前判断：Step 0 通过，Step 1 baseline 已由用户确认通过，Step 2 math parity 已通过；Megatron/ref trainer-side G1 smoke 已通过，维护脚本已落在 `refactor_debugging/g1_plan/run_g1_megatron_ref_smoke.sh`，group-aligned DP split 已有单元测试。下一步是 DP>1 真实 smoke、runtime parity dump，以及 EBFT actor loss 决策。
